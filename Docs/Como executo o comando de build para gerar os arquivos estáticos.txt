Como executo o comando de build para gerar os arquivos estáticos (HTML, CSS, JS) que serão implantados.

Ótima pergunta! A execução do comando de build é a chave para transformar seu código-fonte em um formato otimizado para ser entregue aos usuários. Como mencionei anteriormente, o comando específico dependerá das ferramentas e frameworks que você está usando.
No ambiente JavaScript/TypeScript, o processo de build é geralmente configurado dentro do arquivo package.json do seu projeto, na seção "scripts" . Essa é uma convenção padrão no ecossistema Node.js/npm.

Aqui está como você geralmente executa o comando de build para gerar os arquivos estáticos:

Abra seu Terminal ou Prompt de Comando: Você precisará de uma interface de linha de comando para navegar até o diretório do seu projeto.
Navegue até o Diretório Raiz do seu Projeto: Use o comando cd (change directory) para ir até a pasta onde está o seu arquivo package.json . Por exemplo:
cd /caminho/para/meu/projeto/web

Instale as Dependências (Se ainda não o fez): Antes de executar qualquer script de build, certifique-se de que todas as dependências do seu projeto (as bibliotecas e ferramentas necessárias para o build) estejam instaladas. Você faz isso com:
npm install
Ou, se estiver usando Yarn:
yarn install
Este comando lê o arquivo package.json e baixa tudo o que o projeto precisa para funcionar e ser construído.

Execute o Comando de Build: A maioria dos projetos modernos terá um script de build definido como "build" no package.json . Você o executa da seguinte forma:
npm run build
Ou, se estiver usando Yarn:
yarn build
O que acontece quando você executa npm run build ?

Quando você executa este comando, o script de build configurado no seu package.json será ativado. Ele usará ferramentas como Webpack, Esbuild, Vite, ou o CLI (Command Line Interface) de frameworks como React, Angular, ou Vue para:
Transpilar seu código TypeScript para JavaScript.
Empacotar todos os seus arquivos JavaScript, CSS e outros recursos.
Minificar o código para reduzir o tamanho dos arquivos.
Gerar o HTML final.
Onde estão os arquivos gerados?

Após a conclusão bem-sucedida do comando de build, você encontrará os arquivos estáticos gerados em um diretório específico. Por convenção, este diretório é frequentemente chamado de build , dist (de "distribution") ou public . É este diretório que você informará ao Firebase Hosting como sendo o diretório a ser implantado.
Exemplo de um package.json simples:
{
  "name": "meu-app-web",
  "version": "1.0.0",
  "scripts": {
    "start": "webpack serve --mode development",
    "build": "webpack --mode production"  <-- Este é o comando de build!
  },
  "devDependencies": {
    "webpack": "^5.0.0",
    "webpack-cli": "^4.0.0",
    "typescript": "^4.0.0",
    // ... outras dependências
  }
}
